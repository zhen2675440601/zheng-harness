# 代码规范

本文档定义了本项目的代码提交规范和注释规范，所有成员必须遵守。

---

## 一、Git 提交规范

### 1.1 提交信息语言

**统一使用中文**，包括标题和正文。

### 1.2 提交信息格式

采用 `<类型>(<范围>): <描述>` 格式，参考 [Conventional Commits](https://www.conventionalcommits.org/)：

```
<类型>(<范围>): <标题>

<正文>
```

### 1.3 类型（type）说明

| 类型 | 含义 | 示例 |
|------|------|------|
| `feat` | 新功能 | `feat(agent): 添加任务队列支持` |
| `fix` | Bug 修复 | `fix(runtime): 修复任务超时问题` |
| `refactor` | 代码重构 | `refactor(store): 简化持久化逻辑` |
| `perf` | 性能优化 | `perf(llm): 优化模型调用缓存` |
| `test` | 测试相关 | `test(verify): 添加集成测试` |
| `docs` | 文档更新 | `docs: 更新使用说明` |
| `chore` | 构建/依赖/工具 | `chore(deps): 更新 Go 依赖` |
| `style` | 代码格式 | `style: 格式化代码` |
| `ci` | CI/CD 配置 | `ci: 添加 GitHub Actions` |

### 1.4 范围（scope）

可选，表示修改影响的模块：

- `agent` - Agent 主程序
- `runtime` - 运行时引擎
- `store` - 存储层
- `llm` - LLM 提供商
- `verify` - 验证模块
- `cli` - 命令行工具
- `config` - 配置相关
- `tools` - 工具适配器

### 1.5 标题规则

- 使用中文，简洁明了，不超过 50 字
- 结尾不加句号
- 使用祈使句

### 1.6 正文规则

- 阐述「为什么」修改，而非「做了什么」
- 每行不超过 72 字
- 遇到问题需说明原因和解决方案

### 1.7 示例

```
feat(agent): 添加会话恢复功能

- 支持从 SQLite 数据库恢复中断的会话
- 修复了进程异常退出导致状态丢失的问题
- 新增 --resume 参数用于恢复指定会话

关闭 #123
```

## 二、代码注释规范

### 2.1 语言要求

**所有注释必须使用中文**。

### 2.2 注释类型

#### 2.2.1 行注释

```go
// 计算最终得分
score := calculateScore(items)
```

#### 2.2.2 块注释

```go
// VerifyTaskConfig 验证任务配置的有效性
// 
// 检查项包括：
//   - 超时时间必须为正数
//   - 重试次数不能超过最大限制
//   - 验证策略必须已注册
//
// 如果验证失败，返回具体的错误信息。
func VerifyTaskConfig(cfg *TaskConfig) error {
    // ...
}
```

### 2.3 注释原则

| 场景 | 要求 |
|------|------|
| 公开 API | 必须添加文档注释，说明用途、参数、返回值 |
| 复杂逻辑 | 必须添加注释，解释业务逻辑 |
| 非常规实现 | 必须添加注释，说明为何如此实现 |
| 待完成代码 | 使用 `TODO: <描述>` 标记 |
| 已知问题 | 使用 `FIXME: <描述>` 标记 |
| 常规代码 | 保持代码自解释，不添加冗余注释 |

### 2.4 禁止事项

- ❌ 禁止使用英文注释
- ❌ 禁止使用代码翻译式注释（如 `// i = i + 1` → `// i 自增`）
- ❌ 禁止在注释中遗留临时调试代码
- ❌ 禁止注释掉的代码长期存在

### 2.5 示例

```go
// NewRuntime 创建并初始化运行时实例
//
// 参数:
//   - config: 运行时配置，包含模型提供方、超时设置等
//
// 返回值:
//   - *Runtime: 初始化的运行时实例
//   - error: 初始化失败时返回错误
func NewRuntime(config *RuntimeConfig) (*Runtime, error) {
    if config == nil {
        return nil, errors.New("配置不能为空")
    }
    
    // 初始化模型适配器
    adapter, err := NewModelAdapter(config.Provider)
    if err != nil {
        return nil, fmt.Errorf("初始化模型适配器失败: %w", err)
    }
    
    return &Runtime{
        adapter: adapter,
        config:  config,
    }, nil
}
```

## 三、代码审查 Checklist

提交代码前自查：

- [ ] 提交信息使用中文，格式符合规范
- [ ] 新增代码已添加中文注释
- [ ] 修改的公开 API 已更新文档注释
- [ ] TODO/FIXME 注释已说明负责人和计划时间
- [ ] 无残留的注释代码或调试日志

## 四、自动化工具

项目使用以下工具保证代码质量：

| 工具 | 用途 |
|------|------|
| `gofmt` | 代码格式化 |
| `golangci-lint` | 代码检查 |
| `go vet` | 静态分析 |

提交前请运行：

```bash
make lint
make test
```

## 五、违规处理

- 未使用中文的提交信息将被退回
- 缺少必要注释的代码将被要求补充
- 代码审查发现问题需在 24 小时内修复
